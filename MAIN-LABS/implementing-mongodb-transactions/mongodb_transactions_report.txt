MongoDB transactions are important for maintaining data consistency when multiple database operations must either fully succeed or fully fail together. In this lab, transactions were used to simulate a bank fund transfer between two accounts. Using start_transaction(), both the debit and credit operations were executed as one atomic operation. If an error occurred during the transaction, the changes were rolled back using abort_transaction().

One challenge encountered during the lab was configuring the MongoDB replica set, since transactions require a replica set environment. After initializing the replica set with rs.initiate(), the transaction operations worked correctly. This lab demonstrated how transactions help prevent inconsistent data, especially in financial systems where partial updates could cause serious problems.
